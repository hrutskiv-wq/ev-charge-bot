## Технічні конвенції (обовʼязкові, деталі — docs/PROJECT_CONTEXT.md)

- Python 3.11 · FastAPI · aiogram 3.x · asyncpg із **сирим SQL без ORM** · Alembic
- Єдині екземпляри `bot`/`dp` — з core loader; нових не створювати
- Кожна Alembic-міграція дублюється idempotent-блоком у `create_tables()` — оновлювати ОБИДВА місця
- Будь-яка зміна балансу користувача — ТІЛЬКИ через `update_user_balance()`; ніколи напряму в баланс чи журнал транзакцій
- Мультитенантність білінгу: кожна таблиця і кожен запит — з `operator_id`; тести ізоляції тенантів обовʼязкові
- Секрети — тільки `.env`; ніколи не комітити, не логувати
- Тести — на фейкових зʼєднаннях без живої БД, у стилі існуючих
- Прод (DigitalOcean droplet) агентом напряму не чіпати: локальна розробка → git push → деплой руками

## Правила виконання задач

1. Один промпт = один модуль; після модуля — інструкція запуску + тести. Не генерувати «все одразу»
2. Платіжна логіка, міграції, баланси — пояснюй план ДО написання коду і познач місця для ручного рев'ю
3. Якщо задача суперечить конвенціям вище — зупинись і спитай, а не обходь
4. Дані/цифри в бізнес-файлах — оцінки; не сприймай як факти про ринок без перевірки

## Бізнес-контекст одним абзацом

Соло-засновник, повний фокус, бюджет ~0 → шлях: пілоти → перші платні оператори (590 грн/міс + 3–5% комісія) → реінвест. Kill-критерій білінгу: <5 платних операторів через 3 міс після пілотів → перемикання на ФОП. Довга траєкторія: білінг → smart charging → агрегація гнучкості (VPP).
