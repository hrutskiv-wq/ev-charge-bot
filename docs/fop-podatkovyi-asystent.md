# 💼 ФОП Податковий Асистент — AI-бухгалтер у Telegram

*Бізнес-напрямок: автоматизація податків для 2 млн українських ФОПів. Курс ~42 грн/$.*

---

## Суть продукту

Telegram-бот, який сам рахує ЄП і ЄСВ, нагадує про дедлайни, розпізнає банківські виписки та генерує платіжки з QR-кодом. Аналог QuickBooks/Bench (США) у форматі бота.

**TAM України:** ~$60–100M/рік (2 млн ФОП × $30–50/рік). Навіть 5% ринку = $3–5M/рік виручки.

---

## Архітектура

```
Telegram Bot (aiogram, Python)
        │
Backend API (FastAPI)
        │
├── PostgreSQL — користувачі, доходи, платежі, дедлайни
│     (спільний сервер, база fop_db — див. spilna-infrastruktura-server.md)
├── Claude API — розпізнавання банківських виписок (скрін/PDF → суми доходів)
├── Модуль податкової логіки — ставки ЄП 1/2/3 груп, ЄСВ, ліміти, воєнні пільги
├── Планувальник (Celery + Redis) — нагадування про дедлайни
└── Платежі — LiqPay / WayForPay (підписка)
```

**Ключові технічні рішення:**
- Податкова логіка — окремий модуль з конфігом ставок (вони змінюються — оновлюєте конфіг, не код)
- Розпізнавання виписок: користувач кидає скрін з Приват24/mono → Claude API витягує суми та дати → бот питає підтвердження → записує в книгу доходів
- Генерація платіжок: реквізити податкової по регіонах + QR-код для оплати з банкінгу
- MVP БЕЗ інтеграції з API банків (це +2 міс і складна сертифікація) — додаєте у v2

**Команда:** 1 оператор AI-агентів + 1 консультант-бухгалтер (разово, для перевірки логіки)
**Термін:** 4–6 тижнів з AI-агентом (10–12 тижнів класично)

## Промпти для Claude Code

**Стартовий промпт (дати агенту першим):**
> Ти — senior-розробник. Ми будуємо MVP покроково, модуль за модулем. Стек — як у нашого існуючого проєкту ev-charge-bot: Python 3.11 + FastAPI + asyncpg із сирим SQL (без ORM) + Alembic для міграцій, aiogram 3.x для Telegram (єдині екземпляри bot/dp у core/loader.py). Пиши production-ready код з обробкою помилок і логуванням. Після кожного модуля — коротка інструкція запуску і тести на фейкових зʼєднаннях (без живої БД). Секрети — тільки через .env. Не переускладнюй: MVP, а не enterprise. Не створюй власні Postgres і Redis — підключайся до спільних сервісів з docker-мережі infra_net (див. spilna-infrastruktura-server.md): база fop_db, Redis logical DB 1.

**Промпт 1 — Фундамент:**
> Створи Telegram-бота на aiogram 3.x + PostgreSQL. Онбординг: користувач обирає групу ФОП (1/2/3), ставку ЄП (для 3 групи: 5% або 3%+ПДВ), дату реєстрації ФОП. Схема БД: users (telegram_id, fop_group, tax_rate, region, created_at), incomes (user_id, amount, date, source, description), payments (user_id, type: ЄП/ЄСВ, amount, period, status, deadline). Меню бота: «➕ Додати дохід», «📊 Мої податки», «📅 Дедлайни», «⚙️ Налаштування». Docker-compose для локального запуску.

**Промпт 2 — Податковий модуль:**
> Створи модуль tax_calculator.py. Вхід: група ФОП, ставка, сума доходів за період. Вихід: суми ЄП і ЄСВ до сплати, дедлайни. Усі ставки, мінімалку, прожитковий мінімум і ліміти груп винеси в config/tax_rates_2026.yaml — щоб оновлювати без зміни коду. Додай юніт-тести на кожну групу, включно з крайніми випадками (перевищення ліміту групи — попередження користувачу). ВАЖЛИВО: у коментарях познач кожне місце, де цифри треба звірити з живим бухгалтером.

**Промпт 3 — Розпізнавання виписок:**
> Додай хендлер: користувач надсилає фото/скриншот банківської виписки (Приват24, monobank) → зображення йде в Claude API (модель claude-sonnet-4-6) з промптом, що витягує масив транзакцій {date, amount, description, incoming: bool}. Показуй користувачу знайдені надходження списком з чекбоксами «✅ це дохід ФОП / ❌ пропустити» → підтверджені пишемо в incomes. Обробка помилок: нечитабельне фото → ввічливе прохання перезняти.

**Промпт 4 — Нагадування і платіжки:**
> Додай Celery + Redis. Щоденна задача о 9:00: знайти користувачів з дедлайном сплати через 7, 3 і 1 день → пуш у Telegram з сумою і кнопкою «Отримати платіжку». Платіжка = повідомлення з реквізитами (IBAN податкової з config по регіонах, призначення платежу за шаблоном) + QR-код (бібліотека qrcode) для сканування з банкінгу.

**Промпт 5 — Підписка:**
> Додай монетизацію: перші 30 днів — усе безкоштовно, далі безкоштовно лише дедлайни, а «Мої податки», виписки й платіжки — по підписці 169 грн/міс. Інтеграція LiqPay (checkout + callback-верифікація підписом). Таблиця subscriptions (user_id, status, paid_until). Джентльменський grace-період 3 дні після закінчення.

## Що AI-агент НЕ закриє (робити руками)

1. **Перевірка податкової логіки живим бухгалтером** — обов'язково перед запуском, $100–200 разова консультація. Помилка в ставці ЄСВ вб'є довіру миттєво
2. **Мерчант-акаунт LiqPay/WayForPay** — договір, комісії, узгодження
3. **Security-ревʼю коду з платежами** — $200–300 разово
4. **Маркетинг у спільнотах ФОПів** — скрипти вище, але писати й домовлятися доведеться людям

**Метрика успіху MVP:** 500 платних підписок за 3 міс після запуску.

## Фаза 2: за межами бота (тригер: 1 000+ платних підписок)

1. **Веб-кабінет** (React): історія доходів, звіти, аналітика по місяцях — бухгалтерія зручніша на великому екрані
2. **Пряма інтеграція з API monobank/Приват:** автоімпорт доходів без скрінів виписок — головний stepchange продукту
3. **Подача звітності з ЕЦП** через API податкової — повна заміна бухгалтера для 2–3 груп; чек росте до $10–15/міс (рівень «Таксера», але з AI)
4. **Мобільний застосунок** — останнім: для ФОП-задач веб + бот покривають 90% сценаріїв
