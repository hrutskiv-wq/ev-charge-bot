# 🛍️ AI-Продавець для Інтернет-Магазинів — бот у Instagram/Telegram Direct

*Бізнес-напрямок: автоматизація продажів для 50–100 тис. українських онлайн-магазинів. Курс ~42 грн/$.*

---

## Суть продукту

AI-помічник, який відповідає покупцям в Instagram Direct і Telegram 24/7 (ціна, наявність, доставка), сам збирає замовлення в заявку і передає складні діалоги власнику. Аналог ManyChat/Tidio, але з повноцінним AI і під український ринок.

**TAM України:** ~$30–60M/рік. **Головна перевага:** єдиний з трьох проєктів, що легко масштабується на Польщу/Румунію/ЄС — той самий продукт, ринок ×5–10.

---

## Архітектура

```
Instagram Graph API (webhooks на Direct)  +  Telegram Bot API
        │
Message Router (FastAPI)
        │
├── Claude API + RAG:
│     каталог магазину → embeddings (pgvector) →
│     релевантні товари в контекст → відповідь
├── Логіка ескалації — якщо AI не впевнений або клієнт просить людину → пуш власнику
├── Модуль замовлень — збирає: товар, розмір, адреса НП, телефон → заявка
├── Адмінка (React) — каталог з Google Sheets, історія діалогів, заявки
└── PostgreSQL + Redis (спільний сервер, база shop_db —
      див. spilna-infrastruktura-server.md)
```

**Ключові технічні рішення:**
- Каталог через Google Sheets — власники магазинів уже ведуть облік там, нуль навчання
- «Впевненість» AI: якщо питання поза базою знань — бот чесно каже «уточню у менеджера» і пушить власника. Критично: одна вигадана ціна = втрачений клієнт
- Обов'язково: тумблер «вимкнути бота» для власника в один клік
- Instagram API вимагає верифікації Meta Business — закладіть 2–4 тижні, стартуйте з Telegram-магазинів паралельно

**Команда:** 1 оператор AI-агентів
**Термін:** 6–8 тижнів з AI-агентом (10–14 тижнів класично)

## Скрипти продажів

**Холодний DM Instagram-магазину (писати ввечері, коли вони самі тонуть у діректі):**
> «Привіт! Бачу, у вас активний магазин — напевно, дірект розривається 🙂 Скільки повідомлень "яка ціна?" і "чи є в наявності?" ви обробляєте за день? Ми зробили AI-помічника, який відповідає на це за 5 секунд, 24/7, і сам збирає замовлення в заявку. У магазину [приклад] бот закрив 40% запитів уночі — це замовлення, які раніше йшли конкурентам. 14 днів безкоштовно, налаштую сам за 20 хвилин. Цікаво глянути демо?»

**Обробка заперечення «бот зіпсує спілкування з клієнтами»:**
> «Розумію, це найчастіший страх. Тому бот відповідає ЛИШЕ на те, що знає з вашого каталогу — ціни, наявність, доставка. Все нестандартне одразу передає вам з позначкою. Плюс кнопка "вимкнути" в один клік. Давайте тест на тиждень тільки на нічні години — вранці подивитесь діалоги і вирішите.»

**Стратегія перших клієнтів:** 10 знайомих магазинів безкоштовно за відгуки → кейси з цифрами («закрив 40% запитів уночі») → холодні продажі по 20 DM/день.

## Промпти для Claude Code

**Стартовий промпт (дати агенту першим):**
> Ти — senior-розробник. Ми будуємо MVP покроково, модуль за модулем. Стек — як у нашого існуючого проєкту ev-charge-bot: Python 3.11 + FastAPI + asyncpg із сирим SQL (без ORM) + Alembic для міграцій. Пиши production-ready код з обробкою помилок і логуванням. Після кожного модуля — коротка інструкція запуску і тести на фейкових зʼєднаннях (без живої БД). Секрети — тільки через .env. Не переускладнюй: MVP, а не enterprise. Не створюй власні Postgres і Redis — підключайся до спільних сервісів з docker-мережі infra_net (див. spilna-infrastruktura-server.md): база shop_db, Redis logical DB 2.

**Промпт 1 — Фундамент + Telegram-канал продажів:**
> Створи систему: бізнес-акаунт підключає свого Telegram-бота (передає токен через нашу адмінку) → усі вхідні повідомлення клієнтів йдуть у наш message router (FastAPI, webhook). Схема БД: shops (owner, bot_token, settings, ai_enabled), dialogs (shop_id, client_id, status: ai/human/closed), messages (dialog_id, role, text, ts), orders (shop_id, client_id, items, phone, np_address, status). Роутер: якщо ai_enabled і діалог у статусі ai → відповідь генерує LLM; інакше — пересилка власнику.

**Промпт 2 — Каталог з Google Sheets + RAG:**
> Додай синхронізацію каталогу: власник дає посилання на Google Sheet (колонки: назва, ціна, розміри/варіанти, наявність, опис, фото-URL). Раз на 15 хв тягнемо через gspread → рахуємо embeddings → зберігаємо в pgvector. Функція search_products(query, shop_id) → топ-5 релевантних товарів для контексту LLM. УВАГА: спільний Postgres живе на образі postgres:16 без pgvector — спершу перевір наявність розширення; якщо його немає, НЕ міняй образ прод-БД самовільно: для MVP-масштабу (каталоги до ~1000 товарів) зберігай embeddings у JSONB і рахуй cosine у Python — цього достатньо, перехід на pgvector зробимо окремим узгодженим кроком.

**Промпт 3 — Діалоговий движок з ескалацією:**
> Створи dialog_engine.py: системний промпт для Claude API = «Ти продавець-консультант магазину {name}. Відповідай ТІЛЬКИ на основі каталогу нижче. Ціни й наявність — тільки з каталогу, НІКОЛИ не вигадуй. Якщо питання поза каталогом або клієнт хоче людину — поверни escalate=true» + релевантні товари з RAG + історія діалогу. Відповідь структурована: {reply, escalate, order_intent}. Якщо escalate → статус діалогу human + пуш власнику з останніми 5 повідомленнями. Якщо order_intent → сценарій збору замовлення: товар → варіант → телефон → відділення НП → підтвердження → запис в orders + пуш власнику.

**Промпт 4 — Адмінка:**
> Створи веб-адмінку (React + Vite + Tailwind, автентифікація через Telegram Login Widget): дашборд (діалогів сьогодні, % закритих AI, замовлення), список діалогів з можливістю читати й відповідати вручну, тумблер «AI увімкнено/вимкнено» (глобально і на конкретний діалог), налаштування (посилання на Sheet, тексти привітання, години роботи людини).

**Промпт 5 — Instagram (після верифікації Meta):**
> Додай другий канал: Instagram Direct через Graph API (webhook messages). Той самий message router і dialog_engine, лише інший адаптер вхідних/вихідних повідомлень. Врахуй ліміти: відповідь тільки в 24-годинному вікні після повідомлення клієнта, human_agent tag для ескалацій.
